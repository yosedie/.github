# Contributing to yosedie's Projects

Thank you for your interest in contributing! We welcome contributions of all kinds, including bug reports, feature suggestions, documentation updates, and code enhancements.

To ensure a smooth collaboration, please take a moment to review these guidelines before getting started.

## How Can I Contribute?

### 1. Reporting Bugs
- Search the issue tracker of the specific project to ensure the bug hasn't already been reported.
- If it hasn't, open a new issue using our **Bug Report Template**.
- Provide a clear, descriptive title and include as many details as possible, such as steps to reproduce, expected vs. actual behavior, and environment details.

### 2. Suggesting Enhancements
- Check the issue tracker to see if the feature has already been proposed.
- Open a new issue using our **Feature Request Template**.
- Clearly explain the problem the enhancement solves and why it would be beneficial to the project.

### 3. Submitting Pull Requests (PRs)
- **Fork** the repository and create your branch from `main` or `master`.
- Write clean, documented, and well-tested code.
- Follow the existing code style of the project.
- Open a Pull Request and fill out our **PR Template**.
- Ensure that all tests and checks pass.

## Style Guide & Commits

- Write clear, concise commit messages. We recommend following [Conventional Commits](https://www.conventionalcommits.org/):
  - `feat: add awesome new feature`
  - `fix: resolve crash when opening dashboard`
  - `docs: update setup instructions in README`
- Keep pull requests focused on a single change. If you want to fix multiple unrelated issues, please open separate PRs.

Thank you again for contributing and making the open-source community an amazing place! :rocket:
